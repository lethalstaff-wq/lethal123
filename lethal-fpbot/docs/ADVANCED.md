# Продвинутые техники

Это гайд для пользователей, которые уже освоили базу и хотят выжать
максимум из бота.

## 1. Персональные промокоды для «холодных» лидов

Когда в CRM появляется клиент в сегменте `sleeping` или `churn_risk` —
бот может автоматически:
1. Сгенерить личный промокод на 10% скидку
2. Отправить ему через чат FunPay
3. Отследить использование

Как включить:
```
⚙️ Настройки → 🟢 Воронка продаж
/crm → 💤 Спящие → 📣 Рассылка со скидкой
```

## 2. AI-генератор описаний + Quality Scorer

Pro-фича, которая заменяет час работы на 30 секунд:

```
/lot_gen
→ World of Warcraft
→ 💰 Валюта
→ 1000 Gold
→ 😊 Дружелюбный
```

Получаешь заголовок, описание и SEO keywords. Если quality < 80 —
жмёшь «♻️ Ещё вариант» и бот перегенерит.

Для улучшения существующего лота: нажми «✨ Улучшить имеющийся лот»
и пришли свой текущий текст — бот перепишет лучше.

## 3. Регекс в автоответчике

Обычный автоответчик матчит по substring. Для сложных случаев
используй regex:

В БД поле `trigger_words` для regex-правила — это JSON:
```json
{
  "triggers": ["\\d{3,}"],
  "regex": true,
  "response": "Какое количество нужно?"
}
```

Пример: отвечать на любое упоминание числа ≥ 3 цифр.

## 4. Переменные в ответах

```
Здравствуйте, {buyer}! 👋
{greeting}, по лоту "{lot}" готов дать скидку 5% прямо сейчас.
```

Поддерживаются:
- `{buyer}` — ник покупателя
- `{lot}` — название лота
- `{price}` — цена
- `{time_of_day}` — «доброе утро/день/вечер/ночи»
- `{greeting}` — то же с заглавной

## 5. Варианты ответов через `|||`

```
Привет! Чем помочь?|||Здравствуйте! Что интересует?|||Привет, какой лот?
```

Бот случайно выберет один из трёх. Это снижает подозрение что ты —
бот, и работает как A/B тест.

## 6. Рабочие часы

В JSON правила автоответчика:
```json
{
  "triggers": ["ночь", "спишь"],
  "response": "Да, отдыхаю. Отвечу утром!",
  "working_hours": [0, 8]
}
```

Такое правило сработает только с 00:00 до 08:00 UTC.

## 7. Человеческая задержка

```json
{
  "triggers": ["сколько"],
  "response": "Цена указана в лоте, 500₽",
  "human_delay": true
}
```

Бот будет ждать 1.5-4.5 секунды перед ответом — выглядит как живой человек.

## 8. Smart Pricing стратегии

Каждая стратегия для разных кейсов:

| Стратегия | Когда использовать |
|---|---|
| `always_cheapest` | Демпинг, быстрые продажи, низкомаржинальный товар |
| `top_3` | Баланс видимости и маржи |
| `average` | Стабильная позиция |
| `above_average_5` | Хороший рейтинг, можно держать +5% |
| `above_average_10` | Премиум сегмент, ТОП продавец |
| `below_average_5` | Новый аккаунт, нужна история |
| `below_average_10` | Срочно распродать |

## 9. Плагины

Создай файл `plugins/my_plugin.py`:
```python
def register(manager):
    @manager.on("on_new_order")
    async def my_handler(bot, acc, order):
        print(f"New order: {order.lot_name} for {order.amount}")
```

Плагин загружается при старте или по `/plugins reload`.

Примеры ивентов:
- `on_new_order` — новый оплаченный заказ
- `on_new_message` — новое сообщение в чате
- `on_payment_approved` — тариф активирован
- `on_session_lost` — сессия протухла

## 12. Backup/Restore

```bash
# Ручной backup
python -m cli backup

# Restore (остановить бот)
docker compose down
cp backups/lethal_fpbot-20251201-120000.db data/lethal_fpbot.db
docker compose up -d
```

Авто-backup раз в 24 часа + S3 (если настроено).

## 15. Настройка UX для команды

Если работаете командой продавцов:
1. Один бот, один админ
2. Админ добавляет ФП-аккаунты
3. Включает сервисы
4. Добавляет ID сотрудников в `ADMIN_IDS` — у них будет админка
5. Обычные пользователи не видят админ-команды

## 16. Multi-tenant deployment

Можно запустить несколько инстансов бота на одном сервере с разными
токенами и БД:

```yaml
services:
  bot-client-1:
    build: .
    env_file: .env.client1
    volumes:
      - ./data/client1:/app/data
  bot-client-2:
    build: .
    env_file: .env.client2
    volumes:
      - ./data/client2:/app/data
```

## 17. Авто-бичевка

Схема «бичевки»: дешёвые лоты за 1-5 рублей для постоянного потока
отзывов и прокачки рейтинга.

1. Создай auto_delivery правило с `template` содержащим слово
   `BICHEVKA`
2. Сервис auto_bichevka будет отслеживать и поддерживать
3. После выдачи автоматически просит отзыв

## 18. Cloudflare обход

Если FunPay стал возвращать 403 из-за Cloudflare:

1. Попробуй разные прокси
2. Или смени транспорт на `curl_cffi` для JA3 фингерпринтов:
   ```bash
   pip install curl-cffi
   ```
   И в `funpay/session.py` замени `aiohttp.ClientSession` на
   `curl_cffi.requests.AsyncSession(impersonate="chrome124")`

## 19. Оптимизация под нагрузку

Если клиентов стало 100+:
- Увеличь `HTTP_TIMEOUT` в конфиге до 60с
- Увеличь WAL checkpoint interval
- Добавь SQLite `PRAGMA cache_size = -20000`
- Разбей аккаунты по нескольким процессам через `SHARD_ID` env
